Central retention surface. Archive vs Rubbish Bin per retention value.

- `Archive/` - historically useful material, kept long-term.
- `Rubbish Bin/` - clearly stale, superseded, or low-value material; eligible for purge per `00_System/Rubbish Bin Policy.md`.
- Always mirror the original source path beneath these roots so origin context stays obvious, for example `16_Cleaning/Archive/04_Projects/Project Name/...`.
- Do not delete files outside the narrow rubbish-bin purge path defined in policy.
- Treat final deliverables, contracts, invoices, legal/commercial documents, and client-provided materials as high-retention. Default to `Archive/`, not `Rubbish Bin/`.
- See `00_System/Archive Policy.md`, `00_System/Rubbish Bin Policy.md`, `00_System/Document Lifecycle Policy.md`, and `00_System/Git Preservation Policy.md`.
